This guide walks you through sending a request, saving it, and using environment variables. It also covers importing an existing Postman collection.

## Send your first request

<Steps>
  <Step title="Launch ApiArk and open a collection folder">
    Open ApiArk. You will be prompted to open a folder as a collection, or you can create a new empty directory and open that. The folder you choose becomes the root of your collection — requests you save will be `.yaml` files inside it.
  </Step>
  <Step title="Create a new request tab">
    Press **Ctrl+T** (or **Cmd+T** on macOS), or click the **+** tab button in the tab bar. A new untitled request tab opens.
  </Step>
  <Step title="Enter a URL and choose a method">
    Make sure the method is set to **GET**, then enter the following URL in the address bar:

    ```
    https://jsonplaceholder.typicode.com/todos/1
    ```
  </Step>
  <Step title="Send the request">
    Click **Send**. The response panel shows the status code, response time, and body:

    ```json
    {
      "userId": 1,
      "id": 1,
      "title": "delectus aut autem",
      "completed": false
    }
    ```
  </Step>
  <Step title="Save the request to your collection">
    Press **Ctrl+S** (or **Cmd+S**). Give the request a name — for example, `Get Todo`. ApiArk saves it as a `.yaml` file in your collection folder.
  </Step>
  <Step title="Set up an environment variable">
    Open the **Environments** panel and create a new environment (for example, `Development`). Add a variable:

    | Variable | Value |
    |----------|-------|
    | `baseUrl` | `https://jsonplaceholder.typicode.com` |

    Activate the environment, then update the URL in your request to use the variable:

    ```
    {{baseUrl}}/todos/1
    ```

    ApiArk resolves `{{baseUrl}}` at send time. Your saved YAML file will store the variable reference, not the raw value.
  </Step>
</Steps>

## What the saved YAML looks like

After completing the steps above, the saved request file looks like this:

```yaml
name: Get Todo
method: GET
url: "{{baseUrl}}/todos/1"
assert:
  status: 200
  body.id: { type: integer }
```

This file is plain YAML. You can edit it in any text editor, commit it to Git, and diff it like any other source file.

<Tip>
  Add an `assert` block to a request to get automatic pass/fail feedback every time you send it. The example above asserts that the response status is `200` and that `body.id` is an integer.
</Tip>

## Switching from Postman

If you have an existing Postman collection, you can import it in seconds.

<Steps>
  <Step title="Export from Postman">
    In Postman, open the collection you want to migrate. Click the **...** menu and choose **Export**. Select **Collection v2.1** and save the JSON file.
  </Step>
  <Step title="Open ApiArk">
    Launch ApiArk and open (or create) a collection folder.
  </Step>
  <Step title="Import the collection">
    Press **Ctrl+K** to open the command palette, type **Import Collection**, and select the option. Choose the Postman JSON file you exported.
  </Step>
  <Step title="Done">
    ApiArk converts each request into a `.yaml` file inside your collection folder. Your requests, headers, auth, and body definitions are all preserved.

    <Note>
      ApiArk also imports from Insomnia, Bruno, Hoppscotch, OpenAPI 3.x, HAR, and cURL — not just Postman.
    </Note>
  </Step>
</Steps>
