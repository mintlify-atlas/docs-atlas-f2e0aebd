ApiArk is a desktop API platform built with Tauri v2. It uses ~60 MB of RAM, starts in under 2 seconds, and stores everything as plain YAML files you can commit to Git. No account required, no cloud sync, no telemetry unless you opt in.

## Get started

<Steps>
  <Step title="Download ApiArk">
    Visit the [installation page](/installation) to download the installer for your platform, or use a package manager like Homebrew, Scoop, or APT.
  </Step>
  <Step title="Open or create a collection">
    Launch ApiArk and open an existing folder as a collection, or create a new one. Each collection is just a directory of YAML files.
  </Step>
  <Step title="Send your first request">
    Create a new request tab, enter a URL, and click **Send**. Read the [quickstart guide](/quickstart) for a full walkthrough.
  </Step>
  <Step title="Import from Postman or another tool">
    Already using another API client? Use `Ctrl+K` → "Import Collection" to import from Postman, Insomnia, Bruno, OpenAPI, HAR, or cURL. See [import & export](/integrations/import-export).
  </Step>
</Steps>

## Performance at a glance

| Metric | ApiArk | Postman | Bruno |
|--------|--------|---------|-------|
| RAM at idle | ~60 MB | 300–800 MB | 150–300 MB |
| Cold startup | \<2s | 10–30s | 3–8s |
| Binary size | ~20 MB | ~300 MB | ~150 MB |
| Account required | No | Yes | No |
| Git-friendly | Yes (YAML) | No | Yes (.bru) |

<Note>
  ApiArk is built on Tauri v2 — a Rust backend with the native OS webview — not Electron. This is the reason for the dramatic difference in memory usage and startup time.
</Note>
